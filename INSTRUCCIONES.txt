================================================================
  INSTRUCCIONES DE USO - VOCTOMIX STUDIO
================================================================

REQUISITOS
----------
- Docker, kubectl y minikube instalados
- Python 3 con: python3-gi python3-gi-cairo gir1.2-gtk-3.0


================================================================
  ARRANCAR EL SISTEMA
================================================================

--- OPCIÓN A: Kubernetes (recomendado) ---

  Desde la raíz del proyecto (/home/sonda/Documentos/voctomix):

       ./launch_k8s.sh

  El script arranca minikube, despliega los servicios si es necesario,
  espera a que todo esté listo y abre la GUI automáticamente.

  Para parar: Ctrl+C en la terminal.
  Para parar el clúster al terminar: MINIKUBE_IN_STYLE=false minikube stop


--- OPCIÓN B: Docker local ---

  ./launch_docker_studio.sh

  Para parar: Ctrl+C.

  Si se modifica algún fichero del proyecto, reconstruir antes de arrancar:
    sudo docker compose build


--- OPCIÓN C: Servidor K8s en PC1, operador en PC2 ---

  En PC1:
    cd k8s_escenario && ./start_server_pc1.sh

  En PC2:
    cd k8s_escenario && ./start_operator_pc2.sh
    (si la IP del PC1 no es 192.168.0.42: IP_SERVIDOR=<ip> ./start_operator_pc2.sh)


--- OPCIÓN D: Servidor Docker en PC1, operador en PC2 ---

  En PC1:
    cd docker_escenario3 && ./start_server_pc1.sh

  En PC2:
    cd docker_escenario3 && ./start_operator_pc2.sh
    (si la IP del PC1 no es 192.168.0.42: IP_CEREBRO=<ip> ./start_operator_pc2.sh)


================================================================
  USO DE LA GUI
================================================================

Fuentes (Canal A: F1-F6 / Canal B: botones del panel):
  F1 CAM1  F2 CAM2  F3 CAM3  F4 CAM4  F5 BREAK  F6 INTRO

Modos de composición:
  FS = pantalla completa  |  SBS = lado a lado
  PIP = imagen sobre imagen  |  LEC = conferencia

Transiciones:
  CUT = corte directo  |  TRANS = fundido

Stream Blanker:
  PAUSE = pantalla "Starting soon"
  NOSTREAM = pantalla "Stream offline"
  LIVE = vuelve a emitir


================================================================
  CAMBIAR FUENTES DE VÍDEO
================================================================

Edita k8s/configmap.yaml y cambia la URL o ruta del vídeo:
  CAM1_SOURCE, CAM2_SOURCE, CAM3_SOURCE, CAM4_SOURCE
  BREAK_VIDEO, BREAK_AUDIO, INTRO_VIDEO

Luego aplica los cambios:
  kubectl apply -f k8s/configmap.yaml
  kubectl rollout restart deployment/studio


================================================================
  SOLUCIÓN DE PROBLEMAS
================================================================

GUI no conecta:
  kubectl get pods              ← todos deben estar Running
  kubectl logs deployment/studio -c voctocore

Pods en CrashLoopBackOff:
  kubectl logs deployment/studio -c <nombre-contenedor>

Reiniciar el sistema completo:
  kubectl rollout restart deployment/studio

Reconstruir y subir imagen tras cambios en el código:
  eval $(minikube docker-env)
  docker build -t voctomix:latest .
  docker tag voctomix:latest martin1210235/voctomix:latest
  docker push martin1210235/voctomix:latest
  kubectl rollout restart deployment/studio
