\addcontentsline{toc}{chapter}{Resumen}
{\Large\textsc{\textcolor{nar}{Resumen}}}
\vspace{0.2cm}

La producción de eventos en directo con múltiples cámaras, transiciones, rótulos
y gestión de audio ha requerido históricamente equipamiento de radiodifusión
profesional con costes de adquisición de varios miles de euros y personal técnico
especializado desplazado al lugar del evento. Este modelo resulta inaccesible para
eventos de mediana escala que necesitan producción de calidad.

Este Trabajo Fin de Grado presenta el diseño e implementación de Voctomix~2.0,
una plataforma modular de código abierto para la producción remota de vídeo en
tiempo real. La plataforma se construye sobre GStreamer y Python, siguiendo una
arquitectura cliente-servidor en la que el núcleo de procesamiento
(\texttt{voctocore}) puede ejecutarse de forma remota mientras el realizador opera
la interfaz gráfica (\texttt{voctogui}) desde cualquier equipo de la red. Las
funcionalidades implementadas incluyen cuatro modos de composición de imagen con
transiciones animadas, overlays gráficos dinámicos, un gestor de estados de
emisión con tres modos, sincronización automática del audio y un servicio de
telemetría con publicación en tiempo real en el gestor de colas RabbitMQ.

El sistema se ha contenerizado y desplegado tanto en Docker~Compose como en
Kubernetes con Minikube, integrándose en el proyecto europeo de investigación
CyberNEMO de la UPM, donde opera en tiempo real como plataforma de producción
audiovisual en un entorno de validación activa para diferentes componentes de
ciberseguridad.

La plataforma se ha validado en cuatro escenarios de despliegue: un único equipo,
dos equipos físicos en red local, Docker~Compose y Kubernetes con Minikube. Se
realizaron pruebas de rendimiento y consumo de recursos, mediciones de latencia
de conmutación entre fuentes y de transición entre modos de composición, una
prueba de estabilidad a largo plazo y ensayos de resiliencia ante fallos simulados
de cámara, verificando el funcionamiento en tiempo real del sistema en todos los
entornos evaluados.

El servicio de telemetría se validó de forma independiente, comprobando que los
eventos de producción, el estado de las fuentes y las métricas de consumo de
recursos se publican y almacenan correctamente en RabbitMQ mediante AMQP,
posibilitando su análisis y auditoría posterior.

\addcontentsline{toc}{chapter}{Summary}
{\Large\textsc{\textcolor{nar}{Summary}}}
\vspace{0.2cm}

Live production of events with multiple cameras, transitions, graphics and audio
management has historically required professional broadcast equipment costing
several thousand euros and specialised technical crews deployed on site. This model
is out of reach for medium-scale events that need quality production.

This Bachelor's Thesis presents the design and implementation of Voctomix~2.0, a
modular open-source platform for remote video production in real time. The platform
is built on GStreamer and Python, following a client-server architecture in which
the processing core (\texttt{voctocore}) can run remotely while the director
operates the graphical interface (\texttt{voctogui}) from any machine on the
network. The implemented features include four image composition modes with animated
transitions, dynamic graphical overlays, an output state manager with three modes,
automatic audio synchronisation and a telemetry service that publishes events
in real time to the RabbitMQ message broker.

The system has been containerised and deployed on both Docker~Compose and Kubernetes
with Minikube, integrating into the CyberNEMO European research project at UPM,
where it operates in real time as an audiovisual production platform in an active
validation environment for different cybersecurity components.

Validation was carried out across four deployment scenarios: a single machine, two
physical machines over a local network, Docker~Compose and Kubernetes with Minikube.
Performance and resource consumption tests, switching latency and composition mode
transition measurements, a long-term stability test and resilience trials with
simulated camera failures were conducted, verifying real-time operation of the
system across all evaluated environments.

The telemetry service was validated independently, confirming that production
events, source status and resource consumption metrics are correctly published and
stored in RabbitMQ via AMQP, enabling their subsequent analysis and auditing.
