Este capítulo describe el diseño e implementación de Voctomix~2.0: la arquitectura
del sistema, sus componentes principales y las decisiones técnicas adoptadas. Para
cada módulo se exponen los retos encontrados y las soluciones implementadas.

El trabajo comenzó con una fase de exploración sobre Voctomix~1.0, que permitió
familiarizarse con el pipeline GStreamer, la comunicación cliente-servidor y el
funcionamiento de los composites antes de abordar la versión más reciente.

\subsection{voctocore: núcleo multimedia}
\label{sec:voctocore}

\texttt{voctocore} es el servidor central del sistema: recibe las fuentes de vídeo
entrantes, realiza la mezcla y expone los resultados en distintos puertos TCP. Al
arrancar construye el pipeline GStreamer a partir del fichero de configuración INI,
crea una instancia de \texttt{AVSource} por cada fuente declarada, instancia el
\texttt{VideoMix} y el \texttt{AudioMix}, conecta el \texttt{StreamBlanker} y abre
los puertos de salida.

El puerto~9999 expone el protocolo de control mediante comandos de texto plano como
\texttt{set\_video\_a cam2} o \texttt{set\_composite\_mode pip}. Cada cambio de
estado se difunde a todos los clientes conectados, de modo que la interfaz gráfica
y el servicio de telemetría mantienen siempre una vista coherente del sistema.

\subsection{voctogui: interfaz gráfica de control}
\label{sec:voctogui}

\texttt{voctogui} es el cliente de control: implementado en PyGTK3, proporciona al
realizador la interfaz visual para operar el sistema en tiempo real. Se conecta al
núcleo por TCP y recibe las previsualizaciones de cada fuente en formato JPEG
comprimido (puertos 14000--14005), mientras que el monitor principal recibe la
mezcla en bruto por el puerto~11000.
